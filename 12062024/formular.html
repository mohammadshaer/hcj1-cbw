Einfaches Textfeld

Textfelder werden mit dem input type realisiert. Das Attribute type ermöglicht eine Auswahl von vorkonfigurierten Textfeldern, name ist ein Bezeichner für die Verarbeitung auf dem Server, size gibt die Breite des Textfeldes an und maxlength die Anzahl der Zeichen, die eingegeben werden können.

Labels beschriften Eingabefelder und erhöhen die User-experience. Verbesserung von Zugänglichkeit, Gestaltungsmöglichkeiten für Interaktionsdesign, differenzierte Formatierung, ...

  • Mit value kann ich bei date und datetime-local die Datumsauswahl vorinitialisieren.
  • Datumsformate sind lokalisiert.
  • Es gibt noch weitere zeitbezogene input-types, z.B. month, week. Hier gibt es bei einigen browsern Darstellungsprobleme.
  • Man kann mit JavaScript Attributwerte wie z.B. die aktuelle woche im Jahr mit einem Wert dynamisch vorinitialisieren (z.B. kann man sich an der aktuellen Kalenderwoche orientieren)

Dropdown- bzw. Auswahllisten

Dropdowns werden mit dem select-Tag definiert. Die einzelnen Optionen eines select-Tags werden mit dem option Tag dargestellt. Dieses Option-Tag muss ein value-Attribut haben, damit einen konkreten Wert für die weitere Verarbeitung hat. Die Beschriftung der Optionen steht im innerHTML-Bereich. Dropdowns verwende ich dann, wenn ich eine kleine Menge an sich ausschließenden Optionen habe.
Auswahllisten sind ausgerollte Dropdowns. Mit dem Attribut selected können Elemente vorausgewählt werden.

Radiobuttons

Radiobuttons sind kleine, runde Buttons, die als switch angesehen werden können. Diese lassen sich in Gruppen organisieren. Die Gruppen werden anhand des name-Attributs zugewiesen.
Radiobuttons brauchen ein value-Attribut, um die selektierte Eingabe innerhalb der Gruppe auslesen zu können. Mit dem Attribut checked können Elemente vorausgewählt werden.





Checkboxen

Checkboxen sind UI-Elemente, die eine Mehrfachauswahl zulassen (es können keine, eins oder viele / alle Elemente ausgewählt werden.)
Checkboxen haben ebenfalls ein name-Attribut (für die Gruppe), unterschiedliche Values (value-Attribut) und der type ist "checkbox".




Submit- und Reset-Buttons

Mit einem Submit-Button schicke ich Formular-Daten an einen Server. Bevor das ausgeführt wird, checkt das System, ob alle Formulardaten gemäß Plausibilitäten (ist eine Telefonnummer im gültigen Format? Enthält eine Mailadresse ein @?) und Vollständigkeiten (gem. Pflichtfelder, required-Attribut) die Spezifikation erfüllen. Es gibt viele unterschiedliche input types, um Plausibilitätsprüfungen zu ermöglichen. Es bietet sich an, viele dieser Checks aufgrund einer schnelleren Reaktionszeit und Schonung des Netzwerks clientseitig zu überprüfen.
Mit dem value-Tag des input types submit / reset wird der Button beschriftet. Mit reset setze ich die Eingaben auf das Formular zurück.
